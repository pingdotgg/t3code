# T3 Code — Design Context

Persistent design guidelines for UI work in this project. Generated via teach-impeccable.

## Design Context

### Users

Developers who already use T3 Code on desktop. On mobile they are away from their desk — checking session status, approving actions, reading agent output, and sending quick follow-ups. The job is "stay connected to my coding session without friction," not "replace my IDE on a phone."

### Brand Personality

**Precise, Calm, Capable.**

Voice is direct and technical — no hand-holding fluff. Tone should feel trustworthy and in-control: the user knows their session is alive, reachable, and responsive. Emotional goal: quiet confidence, not excitement.

### Aesthetic Direction

Native companion app — full-screen PWA that feels like it belongs on iOS/Android home screens, not a shrunken desktop site.

- **Visual tone:** Minimal, information-first, developer-native
- **References:** Crisp native companion apps (e.g. Linear mobile for polish and hierarchy; GitHub mobile for utilitarian density). Stay close to existing T3 Code desktop patterns rather than importing a new visual language.
- **Anti-references:** Consumer social apps — no bouncy animations, playful emoji, casual/chatty UI chrome, or generic AI-product aesthetics (purple gradients, glassmorphism everywhere)
- **Theme:** Light and dark equally polished; respect system preference
- **Existing tokens:** DM Sans (UI), SF Mono (code), purple/violet primary (`oklch` hue 264), neutral grays, ~10px base radius, subtle body noise texture, Lucide icons, Tailwind v4 semantic tokens

### Accessibility

- Target WCAG AA contrast for text and interactive elements
- Respect `prefers-reduced-motion`; keep animations purposeful and skippable
- Never rely on color alone for status — pair with icon and label
- Minimum 44px touch targets on coarse pointers (already enforced on buttons via `pointer-coarse`)

### Design Principles

1. **Companion, not clone** — Mobile surfaces what matters on the go (chat, session list, connection status). Do not cram every desktop panel into a phone screen.
2. **Native affordances** — Bottom nav, edge swipe, full-screen standalone mode, safe-area padding. It should feel installed, not bookmarked.
3. **Calm under load** — Reconnecting, pairing, and session state changes should communicate clearly without alarm. Status banners and spinners, not modals and panic.
4. **Precision over decoration** — Tight typography, clear hierarchy, subtle borders and shadows. The purple accent marks primary actions; everything else stays neutral.
5. **Pairing is the front door** — The QR → install → connect flow is the first impression. It should feel secure, fast, and obvious — one path, no dead ends.
