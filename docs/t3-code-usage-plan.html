00 · Independent review (v2 changes)

What the gpt-5.6-sol code review changed.

The v1 plan was reviewed by gpt-5.6-sol with read-only access to the repository; every cited file was re-verified before acceptance. Verdict: product direction and ledger-query shape sound; the persistence design needed restructuring before Phase 1. Four corrections are binding on the rest of this document.

Blocker fixed · durable events, not an ingestion fold

The ledger must be a real projector

In this codebase, projections are registered projectors replaying orchestration_events from a projection_state cursor, with writes transactionally coupled to event appends (ProjectionPipeline.ts:1462–1548, OrchestrationEngine.ts:169–180). v1's in-memory per-turn fold writing rows on turn.completed would be restart-lossy and unreplayable. v2: persist durable model.usage-recorded orchestration events (keeping providerRefs, typed modelUsage, exact cost), then derive the ledger with a standard registered projector. A new projector's empty cursor replays history automatically — this also simplifies backfill.

Misdiagnosis corrected · the adapter was innocent

Cost dies in ingestion, not in ClaudeAdapter

v1 claimed ClaudeAdapter fails to plumb total_cost_usd / modelUsage. Verified false: both completion paths already emit them on turn.completed (ClaudeAdapter.ts:1963–1980, :2036–2050). The loss is downstream — ProviderRuntimeIngestion consumes turn.completed for lifecycle only and discards its usage payload, and buildContextWindowActivityPayload (:596–613) strips providerRefs from persisted usage activities. Phase work moves accordingly. Also confirmed: ThreadTokenUsageSnapshot has no cache-creation field, so "emit the full split" requires a contract extension, and modelUsage needs a typed schema (currently UnknownRecord).

Identity fixed · NULLs defeat uniqueness

Fact IDs, not composite keys

v1's dedup key (environment, thread, turn, model) fails twice: SQLite UNIQUE ignores NULL components (interrupted turns have no turn_id), and it omits the provider-native session id that validation proved essential (concurrent Codex sessions on one T3 thread inflated sqlite replay 12×). v2: a stable fact_id hashed from provider + native session + native turn/message id + model + kind, with explicit upsert precedence (live > transcripts > events) and a stale flag for lifecycle-rejected completions. Per-model cost and turn-level totalCostUsd are stored as separate fact kinds so totals reconcile instead of double-counting.

Semantics added · lifecycle, revert, terminal paths

Usage survives what conversations don't

Existing projectors prune on thread.reverted (ProjectionPipeline.ts:851–980); the usage projector deliberately does not — consumed tokens are immutable billing history. Thread deletion keeps facts and redacts linkage. Finalization listens to turn.aborted (Codex emits it: CodexAdapter.ts:788–797), session exit, and steering-superseded turns — not just turn.completed. Backfill leaves schema migrations (which run during SQLite init) and becomes a versioned, resumable server-side job; transcript parsing is opt-in and never runs at startup. Pricing moves to versioned, effective-dated data in shared runtime code, never packages/contracts.

Confirmed feasible by the review
  • Multi-environment fan-out works with the existing client registry (EnvironmentRegistry.run is environment-scoped) — but it must lease the environment catalog, not just currently-live connections, with bounded concurrency and per-environment timeouts.
  • One dashboard-shaped usage.query RPC (named bucket sets per dimension) avoids four round-trips per page load; it needs a RPC_REQUIRED_SCOPE read-scope entry like every other method.
  • Grouping timezone comes from the requesting client (IANA), attribution at observation time; DST handled by local-calendar bucketing.
  • Sizing revised: the "scrappy 2–3 day" v1 cut is retired — it would have shipped an unreplayable table. Accounting correctness first; UI was mostly done in mock form anyway.
01 · Product decision

Index once; query the ledger.

The dashboard should never scan gigabytes of JSONL or rotated provider logs on page load. Each T3 environment owns a compact usage ledger in its existing SQLite state. The client asks every connected environment for aggregate rows and merges those rows locally.

Default scope

T3 Code only

Reliably attributable to T3 provider sessions, threads, projects, subagents, and environments.

Optional scope

All local agent activity

Opt-in indexing for Codex, Claude, OpenCode, and future sources launched outside T3.

Placement

Top-level /usage

A destination beside Settings, not a settings panel and not a thread-level activity view.

Do
  • Persist normalized, idempotent usage records.
  • Keep exact provider cost separate from API-equivalent estimates.
  • Return aggregate, coverage, pricing, and freshness metadata.
  • Show partial cross-environment results when a host is offline.
Do not
  • Require a separately installed ccusage binary.
  • Store usage rows in thread activities or conversation snapshots.
  • Call cache tokens “input” without separating their semantics.
  • Publish durable usage aggregates to the cloud relay.

03 · Metric contract

Define the numbers before drawing charts.

Token fields differ across providers. The normalized contract must make inclusion and subset relationships explicit so totals remain comparable.

Uncached input Input tokens that were not served from a provider cache.
Cache read Reused input tokens. Kept separate because these can dominate totals.
Cache write Tokens written into a prompt cache, including Claude cache creation.
Output All generated output. Reasoning output is included here.
Reasoning output A visible subset of output, never added to the total twice.
Total tokens Uncached input + cache read + cache write + output.
Model calls Positive native usage records—not turns and not tool calls.
T3 turns User-initiated turns. A turn can produce many model calls and subagents.
Time of day Turn-start distribution for T3 scope; call distribution for all-local scope.
Do not report “hours spent” from provider logs. Logs reveal activity points, not foreground app duration. A future client-presence signal could add actual active time, but it must be named and stored separately.
04 · Architecture

One normalized path for live and historical usage.

Live adapters emit provider-neutral usage events. Backfill adapters produce identical record identities, so the SQLite uniqueness boundary de-duplicates live and imported data automatically.

Canonical event

model.usage-recorded

A single provider-neutral usage contract. Context-window events remain dedicated to UI occupancy and compaction behavior. 

ProviderUsageRecord {
  nativeRecordKey, provider, providerInstanceId,
  threadId?, turnId?, projectId?, providerSessionId?,
  model, occurredAt, requestCount,
  uncachedInputTokens, cacheReadInputTokens,
  cacheWriteInputTokens, outputTokens,
  reasoningOutputTokens,
  providerCostMicros?, serviceTier?,
  origin: "t3" | "external",
  accuracy: "exact" | "derived" | "estimated"
}
Idempotency boundary

Stable native identity

  • Claude: session ID + assistant message UUID
  • Codex: session ID + monotonic cumulative usage state
  • OpenCode: provider session/message ID
  • ACP: session + turn + response ordinal
  • Canonical fallback: T3 provider event ID
Live ingestion and later backfill insert the same key. A unique index prevents double counting without source-specific cleanup passes.

06 · Pricing and fleet

Aggregate environments, not SSH machines.

T3 already knows its connected execution environments and can run an RPC in any one of them. That catalog is the product-native aggregation boundary and works for local, SSH, bearer, and relay-backed connections.

usage.query

Request

UTC range, IANA timezone, bucket granularity, scope, environment filters, and optional project grouping.

Response

Totals, daily buckets, model rows, hour/weekday distributions, environment/source breakdown, coverage, indexed-through time, errors, and pricing basis.

Failure model

Partial success by default

If one environment is offline, show available totals and “3 of 4 environments reporting.” Older environments without usageReporting show “Update required” instead of failing the page.

connected stale unsupported relay transport
Cross-environment dedupe

Dataset fingerprints

Multiple T3 servers on one physical host can see the same provider directory. Hash a stable host identity plus canonical source root; return per-source rollups and select the freshest duplicate dataset before summing.

Authorization

Read-only and already scoped

Use the existing orchestration:read scope: it already grants access to more sensitive thread snapshots. Manual reindex can require orchestration:operate.

Cost is not billing. Display “API-equivalent estimate,” use provider-reported cost where available, bundle a pinned offline pricing snapshot, preserve service tier, and label unknown custom models “Unpriced” rather than $0.00. Support import of ccusage-compatible pricing overrides.

08 · Delivery sequence

Ship in six reviewable slices.

Establish accounting correctness before UI polish. Each slice leaves a usable, testable boundary and avoids bundling provider parsing, fleet aggregation, and charting into one risky change.

Semantics, durable events, and projector
Add packages/contracts/src/usage.ts (typed facts: cumulative vs interval, cache read vs write, native identity, reset generation; typed modelUsage), emit durable model.usage-recorded orchestration events from ingestion, and register a usage projector with its own projection_state cursor. Schema migration creates tables only. Tests: live-vs-replay equivalence, restart, duplicates, abort, steering, revert retention, deletion redaction.
Codex and Claude live capture
Preserve providerRefs, per-model usage, and exact cost through ingestion (ClaudeAdapter already emits them on turn.completed); finalize on turn.aborted, session exit, and steering-superseded paths, not completion alone. Extend the snapshot contract with cache-creation tokens.
Backfill and pricing
Versioned, resumable server-side backfill job (never inside migrations, no second DB writer): event replay via the projector's empty cursor, then provider-log parse, then opt-in transcript enrichment — same fact IDs, explicit upsert precedence. Versioned, effective-dated pricing catalog in shared runtime code; estimates computed at query time, never stored as truth.
RPC and aggregation
Add usage.query, environment capability negotiation, client-runtime query atoms, partial errors, timezone grouping, and dataset de-duplication.
Web and desktop Usage page
Add the route, sidebar affordance, summary cards, accessible SVG/CSS charts, model and environment tables, coverage states, filters, and empty/error states.
Coverage and mobile
Add OpenCode and ACP capture, all-local opt-in, cached offline aggregates, responsive mobile UI, and the mobile integrated verification pass.

10 · Risks and safeguards

Be honest where inference stops.

The largest risk is not missing data; it is silently presenting duplicate, cumulative, unpriced, or differently defined data as a single exact total.

Cumulative counters double-count. Use monotonic totals, stable native keys, and explicit reset rules.
Live and backfill overlap. Enforce native record uniqueness independent of ingestion source.
Two environments share one provider home. Return per-source rollups and dedupe hashed dataset fingerprints.
Provider formats drift. Version parsers, use golden fixtures, surface importer errors, retain cursors.
Custom models have no public price. Show “Unpriced,” report coverage, and support compatible overrides.
Historical thread deletion changes joins. Persist stable project/thread IDs in usage rows; clear usage separately.
Raw logs contain private content. Whitelist fields during streaming parse; store no prompts, tools, or cwd.
Estimated cost looks like a bill. Label API-equivalent cost, basis, pricing version, tier, and unknowns.
